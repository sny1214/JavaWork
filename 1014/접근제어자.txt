접근제어자(access modifier) 는 클래스 및 객체의 멤버를 
사용할수 있는지 없는지를 규정한 예약어를 말한다.

1. 클래스 선언시 사용하는 접근제어자(default, public)
   (1) 클래스의 접근제어자가 default이므로 현재 패지키에서만 사용할수 있다.
   class A{
     ...
   }
   
   (2) 클래스의 접근제어자가 public이므로 다른 패키지에서도 사용할수 있다
       단, import 시켜서 사용한다.
   public class A{
     ...
   }
 
2. 클래스의 멤버(멤버변수,메서드)의 접근제어자
	class A{
	  int a;              //default 접근제어자 -- 같은 패키지에서 외부에서 자유롭게 접근
	                      //                   다른 패키지에서는 외부에서 접근할 수 없다. 
	  private   int b;    //private 접근제어자 -- 같은 패키지나 다른패키지 모두에서
	                                           외부에서는 접근할 수 없다.
	  protected int c;    //protected 접근제어자 -- 상속과 관련된 접근제어자
	  public    int d;    //public 접근제어자 -- 같은 패키지, 다른 패키지 모두에서
	                                           외부에서 접근 가능
	   ....
    }
   
   (1) private - 같은 패키지이건 다른 패키지이건 외부에서 접근할 수 없다.
   (2) default - 아무것도 사용하지 않은것
                같은 패키지에서는 내.외부에서 자유롭게 사용
                다른 패키지에서는 외부에서 접근할 수 없다.
   (3) protected - 상속된 하위클래스에서만 패키지가 달라도 자유롭게 접근
   (4) public - 같은 패키지,다른 패키지 모두에서 외부에 자유롭게 접근
       
   
   일반적으로 데이타를 보호하기 위해 멤버변수는 private 사용을 권장하고
   메서드는 public을 주로 사용한다.
