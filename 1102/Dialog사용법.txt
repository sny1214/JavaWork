대화상자는 단독으로 실행될수 없고 Frame이나 다른 대화상자를 부모로 해서 작동할 수 있다.
  - 단독으로 생성할 수 없고 반드시 생성자에 Frame/Dialog가 있어야 한다.
  
대화상자(Dialog) : 보조창 역활을 수행하여
   1. 사용자의 확인/취소 등의 확인작업
   2. 사용자의 입력 대체창
   3. 다양한 설정창
   4. 찿기/바꾸기등의 보조창 등으로 많이 사용된다.
   
Component
  Container
     Window
        Frame   ===> 메인창 역활(단독실행 가능)
        Dialog  ===> 보조창 역활(단독실행 못함, 반드시 부모 윈도우가 있어야 한다)
        
Frame과 Dialog는 사용법의 95%이상 동일하다.
 
대화상자의 종류 :
 1) 모달 대화상자(Modal Dialog) : 부모창에 의해 대화상자가 나타난후 
          대화상자가 종료하기 전에는 부모창으로 촛점(focus)가 이동하지 못하는 대화창
 2) 모달리스 대화상자(Modaless Dialog) : 부모창과 대화상자에 focus가 자유롭게 이동할 있는 대화창

단, 모든 대화상자는 부모창이 소멸하면 자동으로 대화창도 소멸된다.          
 
우리가 대화상자를 이용할때는 Dialog를 상속받아 User Dialog를 만들어서
화면구성 및 모든 이벤트처리를 Frame과 동일한 방식으로 처리해야 한다.
